1. 概述

  1. 正则表达式(Regular Expression)的用途:
    用于文本的复杂操作，是一种强大而灵活的文本处理语言，大部分编程语言、数据库、开发环境都支持正则表达式
  2. 使用正则表达式的流程
    • 分析所要匹配的数据，写出测试用的典型数据
    • 在工具软件中进行匹配测试
    • 在程序中调用通过测试的正则表达式

2. 正则表达式语法

  1. 普通字符
    字母、数字、汉字、下划线、没有特殊定义的标点符号都属于普通字符，表达式中的普通字符在匹配字符串时匹配与之相同的一个字符
  2. 转义字符
  3. 标准字符集合
    • 能够与"多种字符"匹配的一个字符，区分大小写，大写则是不匹配对应的多种字符的一个字符
  4. 自定义字符集合
    • []方括号匹配方式，能够匹配方括号中的任意一个字符
    • 正则表达式中的特殊符号，被包含到中括号中则会失去特殊意义，处理"^"和"-"
    • 标准字符集合除小数点外，如果被包含于中括号，自定义字符集合将包含该集合
      e.g. [\d.\-+] - 匹配数字、小数点、"+"、"-"
  5. 量词(Quantifier)
    • 修饰匹配次数的特殊符号
    • e.g.1 匹配4位数字: \d\d\d\d === \d{4}
    • e.g.2 匹配4位"字母+数字": ([a-z A-z]\d){4}
    • 贪婪模式与非贪婪模式
      • 贪婪模式: 匹配字符越多越好,默认为贪婪模式
        e.g. \d{3,4} - "1234"字符被匹配结果为"1234"
      • 非贪婪模式: 匹配字符越少越好,修饰匹配次数的特殊符号后再加一个?即可以设置非贪婪模式
        e.g. \d{3,4}? - "1234"字符被匹配结果为"123"
  6. 字符边界
    • 字符边界匹配 的不是字符而是位置，匹配符合某种条件的位置
    • e.g.1 ^a - "abcdaefga"该字符第一个"a"被匹配
    • e.g.2 b$ - "babcdefgb" 该字符最后一个"b"被匹配
    • e.g.3 abc\b - "abc abc2 abc111 222abc" "abc"是单词边界的被匹配
  7. 匹配模式
  8. 选择符和分组
    反向引用(\nn):每一对()会分配一个编号,使用()的捕获根据左括号的顺序从1开始自动编号;通过反向引用可以对分组已捕获的内容进行引用
    e.g. e.g. ([a-z]{2})\1 - goto gogo toto dodo todo
  9. 零宽断言(环视)
    • 只进行子表达式的匹配，匹配内容不计入最终匹配结果，是零宽度的
    • 这个位置应该符合某个条件，判断当前位置的前后字符是否符合指定的条件，但不匹配前后的字符，是相对位置的匹配
    • 正则表达式匹配过程中，如果字表达是匹配的是字符内容而非位置，并被保存到最终匹配结果中，那么就认为这个子表达式是占有字符的；如果子表达式匹配的仅仅是位置或者匹配的内容并不保存到最终的匹配结果中，那么就认为这个子表达式是零宽度的，占有字符还是零宽度，是针对匹配的内容是否保存到最终匹配结果而定的
    • e.g. [a-z]+(?=ing) - going doing eating

3. 正则表达式应用

  1. 电话号码的验证
    • 规则
      • 电话号码由数字和"-"构成
      • 电话号码为7位到8位
      • 如果电话号码中包含有区号，那么区号为三位或四位，首位是0
      • 移动电话号码为11位
      • 11位移动电话号码的第一位和第二位为"13" / "15" / "17" / "18" / "19"
    • 表达式: (0\d{2,3}-\d{7,9})|(1[35789]\d{9})
  2. 邮箱的验证
    • 规则
      • 用户名: 由字母、数字、中划线、下划线组成
      • @
      • 网址: 字母、数字组成
      • 小数点: .
      • 组织域名: 2-4位字母组成
      • 不区分大小写
    • 表达式: [\w\-]+@[a-z0-9A-Z]+(\.[a-zA-Z]{2,4}){1,2}

4. 在Java中使用正则表达式

  1. 尝试将整个字符串进行匹配 
    //"abcde&&edcba"是否符合指定的正则表达式"\w+";
//正则表达式对象("\"要变为"\\")
Pattern pattern = Pattern.compile("\\w+");
//匹配对象
Matcher matcher = pattern.matcher("abcde&&edcba");
//尝试将整个字符序列与该模式匹配
boolean result = matcher.matches();
//打印匹配结果
System.out.println(result);
  2. 扫描字符序列,查询符合的子字符段 
    Pattern pattern = Pattern.compile("\\w+");
Matcher matcher = pattern.matcher("abcde&&edcba");
//扫描字符序列,查找与该模式匹配的下一个子序列
while (matcher.find()) {
//打印匹配的字符内容
    System.out.println(matcher.group());
}
  3. 分组输出匹配结果 
    Pattern pattern = Pattern.compile("([a-z]+)([0-9]+)");
Matcher matcher = pattern.matcher("aa232**ssd445*sds223");
while (matcher.find()) {
    //分组显示匹配的字符串
    System.out.println(matcher.group());
    System.out.println(matcher.group(1));  //([a-z]+)
    System.out.println(matcher.group(2));  //([0-9]+)
}
  4. 字符串替换操作 
    Pattern pattern = Pattern.compile("([0-9]+)");
Matcher matcher = pattern.matcher("aa232**ssd445*sds223");
//替换操作
String str1 = matcher.replaceAll("#");
System.out.println(str1);
  5. 字符串分割操作 
    //字符串分割操作
String str = "aa232**ssd445*sds223";
String [] arrs = str.split("\\d+");
System.out.println(Arrays.toString(arrs));
